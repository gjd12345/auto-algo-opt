## 0. 一句话结论

Codex 的**数字可信**(我逐格复算一致),但其「这批没有真实在线 LLM rerank」的**口径是错的**——代码与数据都证明 reranker 实际驱动了 545/605 的选卡注入,`enabled=False` 只是日志假象。真正的因果障碍不是「reranker 没跑」,而是**没有对照臂**。此外我自选了 7 个口径,得出一个连贯故事:**三个问题的"难度画像"截然不同**,直接决定算力分配与论文能讲什么。

## 1. 对 Codex 结论的复核

### 1.1 可信(逐格复算一致)
| 项 | 复核结果 |
|---|---|
| 覆盖 | 605/605 映射成功,0 缺失 |

| 单个最优来源 | bp/tsp=gen8、cvrp=gen16 |

### 1.2 必须纠正的口径(代码 + 数据双证)
Codex 的 Q6 与修正计划写「605 条 `rag_rerank_enabled=False` → 这批没有真实在线 LLM rerank / 不能证明 reranker」。**前半句错。**

- **代码** `eoh_rag/experiments/rag_context_builder.py`:
  - L340–353:llm 模式 `llm_selected = llm_rerank(...)`,`if llm_selected: retrieved = llm_selected` → reranker 选中的卡**直接成为实际注入卡**。
  - L450:`rag_rerank_enabled` 在 llm 模式下被**硬编码为 False**,与是否生效无关。
- **数据**:605/605 都有 rerank 延迟(中位 19.8s / 最大 63s)与 reasoning;**545 个 run 实际注入卡与 reranker 选择 100% 一致**,60 个为空选回退。候选池 5–6 张(有真实取舍)。

**纠正后**:reranker **确实是这批的在线选卡机制**;结论方向(不能下因果)对,但理由应改为「**缺对照臂**」。含义:蒸馏 target(`rag_llm_rerank_selected` = 实际注入策略)**现在就绪**,无需重跑;补因果只需加一个匹配对照臂。

## 2. 自选口径的分析(增量,Codex/deepseek 未做)

所有数字来自 605 raw summary + 轨迹 CSV。

### 2.1 收敛动力学:BP 晚熟,TSP/CVRP 前载
逐代 median best-so-far 提升 + 相邻代边际:

| 问题 | g0 | g2 | g4 | g8 | g12 | g16 | 边际形态 |
|---|---|---|---|---|---|---|---|
| bp_online | -0.1% | -0.1% | 1.4% | 6.6% | 22.4% | 26.9% | 前 2 代≈0,**边际递增**,g8 后陡升 |
| tsp_construct | 0.5% | 3.2% | 3.9% | 5.2% | 5.5% | 5.7% | g0→1 +1.8%/代,g3–4 见顶 |
| cvrp_construct | 1.8% | 3.6% | 4.3% | 4.9% | 5.1% | 5.5% | g0→1 +1.5%/代,g3–4 见顶 |

→ **BP 是罕见的"晚熟+边际递增"**:多数搜索边际递减,BP 却在 gen5–6 仍在加速,gen8 远未收敛。TSP/CVRP 是典型"前载+快速平台"。

### 2.2 分代 ROI(质量 ÷ 成本)
单 run 运行时长中位:**gen8 = 522s,gen16 = 957s(1.83×)**。

| 问题 | median 提升 gen8→16 | 倍数 | >5% 率 gen8→16 | gen16 是否值 |
|---|---|---|---|---|
| bp_online | 6.6% → 26.9% | **4.1×** | 54%→80% | **非常值**(1.83× 成本换 4× 质量) |
| tsp_construct | 5.2% → 5.7% | 1.1× | 54%→73% | 质量不值;仅抬达标率 |
| cvrp_construct | 4.9% → 5.5% | 1.1× | 42%→67% | 质量不值;主要抬达标率 |

→ **算力建议**:BP 用 gen≥16;TSP/CVRP gen8 性价比已足(若目标是"更多 run 过 5% 门槛"再上 gen16)。

### 2.3 LLM 生成难度(样本失败率)
每代生成即不可用(编译/校验失败)的样本比例(run_log 中位):

| 问题 | gen8 | gen16 |
|---|---|---|
| bp_online | **45.0%** | 47.7% |
| cvrp_construct | 18.3% | 20.4% |
| tsp_construct | 11.7% | 11.6% |

→ **BP 近半候选无效**,远高于 TSP/CVRP。BP 的 `score` 函数空间(numpy 数组、数值稳定性)对 LLM 最难。这与"BP 晚熟"一致:有效候选稀疏 → 需要更多代才能积累出好解。**工程含义**:BP 的算力有近一半耗在无效样本上,是 self-repair / 更强 prompt 约束的首要目标。

### 2.4 种子主导 vs 搜索主导
gen0 提升与最终提升的相关系数:

| 问题 | r | 解读 |
|---|---|---|
| cvrp_construct | **+0.62** | **起点主导**——起得好基本就赢,搜索空间浅 |
| tsp_construct | +0.20 | 搜索主导(弱起点相关) |
| bp_online | +0.15 | **搜索主导**——起点几乎不预测终点,全靠进化 |

→ **含义**:CVRP 最该投入在**更好的种子/初始卡**(收益天花板由起点定);BP 最该投入在**更长/更强的搜索**(起点无用)。

### 2.5 Reranker 政策画像(蒸馏 target 是什么)
545 条 reranker reasoning 的概念关键词命中率:

| 概念 | 命中率 |
|---|---|
| 搜索方向 | 76% |
| 互补 | 74% |
| 避免重复 | 53% |
| 探索 | 35% |
| 多样性 | 30% |
| 种群特征 | 22% |
| 覆盖 | 10% |
| 收敛 | **0%** |

→ reranker 的政策**压倒性是"互补/多样/避免重复"(相对现有种群求异)**,几乎不谈收敛,只有 22% 显式提种群特征。**phase-2 小模型应学的是**:一个**多样性/互补驱动、以种群为条件**的卡片选择器。(注:此为中文关键词计数的粗画像,非语义解析。)

### 2.6 选卡集中度:CVRP 塌缩、BP 丰富
非空选择 run 中,最常见卡片组合的占比:

| 问题 | 最常组合 | 占比 | 组合种类数 |
|---|---|---|---|
| cvrp_construct | far_first + sweep | **92%** | 3 |
| tsp_construct | farthest_insertion + two_opt | 67% | 5 |
| bp_online | best_fit + first_fit | 41% | **13** |

→ **CVRP/TSP 的 reranker 选择近乎常量**(尤其 CVRP 几乎只选一种);**BP 选择最丰富**。

### 2.7 蒸馏价值定位(综合 2.5/2.6 + 结果)
- **BP**:选择多样(13 组合)且与结果强耦合(含 `funsearch+sqrt_exp` 的 run median ~27% vs 含 `first_fit+best_fit` 的 ~4%)。→ **这里的选择策略"可学且值得学"**。
- **CVRP/TSP**:选择近乎常量(92%/67% 同一组合)→ 一个近似"常数策略"就能复现,**可学的信号很少**。

→ **对论文主线的直接指导**:小模型 selector 的价值验证应**优先在 BP** 上做(选择空间大、结果敏感);CVRP/TSP 更像"选择已收敛",适合作为"策略稳定"的对照。

## 3. 能 / 不能支撑的主张(scope)

- ✅ **能**:进化有效性 + 三问题差异化收敛画像;算力/成本分配建议;reranker 政策画像与蒸馏就绪度;选卡结果的观察性假设。
- ⚠ **观察性(仅假设)**:选卡 lift、种群特征相关——选卡未随机化 + 收敛混淆,不可写因果。
- ❌ **不能**:RAG 整体因果收益(单臂 literature_rag)、在线 reranker 因果增量(**无对照臂**,注意:不是"没跑")、逐代选卡因果归因(rag_trace 为 run-level,无逐代重选 instrumentation)。

## 4. 建议的下一步(据本分析)

1. **论文主线优先在 BP 验证小模型 selector**(§2.7):选择多样 + 结果敏感,蒸馏 target 已就绪(`rag_llm_rerank_selected`,545 条)。
2. **补对照臂做因果**(§1.2):固定其余条件,加 no-rerank / 换 reranker 的匹配臂,才能证 reranker 因果增量。
3. **算力重分配**(§2.2/2.3):BP 上 gen≥16 且优先治理 45% 无效样本(self-repair/更严 prompt);TSP/CVRP 保持 gen8。
4. **CVRP 投种子、BP 投搜索**(§2.4)。
5. 逐代选卡 instrumentation(记录每代重选)以支撑 Q1/Q4 时序因果。
