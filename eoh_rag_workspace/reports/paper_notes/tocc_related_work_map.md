## 关键差异化边界

| 方法 | 改变什么 | TOCC 与其关系 |
|------|----------|---------------|
| **CoEvo-AHD** | 算子如何实现（给工具） | **正交可组合**：cards + tools |
| **A2DEPT** | 搜索空间大小（开放式 AAD） | **思想借鉴**：自适应调度→card selection 学习 |
| **HeurAgenix** | 启发式从哪来（进化+选择） | **路径不同**：重路径(代码) vs 轻路径(卡片) |
| **EoH** | 组件参数（进化框架内调优） | **TOCC 的基础**：TOCC 在 EoH 之上加 card selection |
| **FunSearch** | 算法发现（岛屿演化） | **同族**：都是 LLM + 进化，TOCC 加了知识注入 |
| **CO-Bench** | 评测标准（基准测试） | **评测参考**：可作为 TOCC 的评测框架 |
| **HeuriGym** | 评测指标（QYI） | **指标借鉴**：QYI = Quality × Yield |

## 一句话定位

```
CoEvo-AHD: 给算子提供计算工具
A2DEPT:    扩大搜索空间
HeurAgenix: 进化可执行代码
TOCC:      给 LLM 提供策略知识（改变生成分布）
```

## 可组合场景

```
TOCC + CoEvo-AHD: LLM 同时获得策略知识（cards）和计算工具（env primitives）
TOCC + A2DEPT:    cards 作为开放式搜索的方向引导
TOCC + HeurAgenix: cards 作为种子启发式的知识增强
```
