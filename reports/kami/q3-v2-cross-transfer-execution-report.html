02 · Delivered System

已交付工程

运行契约

新增不可变 RunSpec，将 suite、problem、arm、generation、repeat、seed、run key 与输出目录绑定。矩阵按 seed → problem → arm 展开，problem-scoped arm 按实际数量计数；summary 与 run index 记录 seed。

隔离与安全

  • Q3 固定 n_processes=1，禁用 pool、outcome 与 previous-run chain。
  • Provider 日志只保留名称、endpoint host、模型和 key 是否存在。
  • 429 与额度耗尽分别归类，fixed 温度保持 Provider 默认。
  • 原始响应、population、trace、数据集与认证路径不进入 Git。

Core held-out

TSP 与 CVRP evaluator 已删除 0.0 占位，改为真实构造路线、验证可行性并输出 cost、optimum 与 relative gap；失败使用结构化错误。PoolAPI.snapshot 稳定排序、深拷贝、problem 隔离且零写入。每代 population 生成 AST 多样性摘要。

冻结策略与清单

Q3 固定 10 seeds、三臂和四个算子。Cross 固定 BP/TSP/CVRP、两臂和 5 seeds；两臂共享同一 605-run 快照。12 条中文抽象策略与 transfer map 已冻结，卡数和 2,500 字符预算一致。
